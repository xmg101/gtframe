# 自动化游戏测试框架 — Web UI 设计

> 设计日期：2026-05-31

## 概述

为 gtframe 增加可视化 Web UI 界面，让 QA 团队可以通过浏览器操作测试框架的全部核心功能。使用 FastAPI 后端 + 简单前端页面，`gtframe web` 命令启动。

## 技术选型

| 组件 | 选型 | 理由 |
|------|------|------|
| 后端框架 | FastAPI | Python 原生，异步支持，自动生成 API 文档 |
| 前端 | HTML + CSS + JS (无框架) | 零依赖，无需构建工具，对非专业用户友好 |
| 启动方式 | `gtframe web` | 复用已有 CLI 入口 |
| 端口 | 默认 8765 | 避免常见端口冲突 |
| 模板 | Jinja2 (FastAPI 内置) | 服务端渲染，不写独立的 SPA |

## 页面列表（5 个）

### 1. 仪表盘 (Dashboard)

进入 Web 的首页，展示：
- 4 个统计卡片：用例总数、今日执行次数、通过率、设备在线数
- 快速操作按钮：运行所有用例、查看最近报告、刷新设备
- 最近执行记录列表：显示最近的 5-10 次测试结果（名称、状态、时间）

### 2. 运行测试 (Run)

核心功能页面，三步操作流程：

1. **选择用例** — 文件列表展示 `cases/` 目录下的 .yaml 文件，复选框多选，搜索过滤
2. **选择设备** — 下拉菜单列出已连接设备，绿色圆点表示在线
3. **运行 & 实时输出** — 点击"运行"按钮，异步执行，黑色终端风格日志面板实时滚动显示

日志面板每行显示：
- 时间戳
- 步骤序号和 action 名称
- 状态（通过 ✓ / 失败 ✗ / 进行中 ⏳）
- 耗时

运行结束后自动显示摘要（总通过数/总数/总耗时）。

### 3. 测试报告 (Reports)

历史报告列表：
- 搜索和状态过滤
- 每条报告显示：名称、时间、通过/总数、耗时、状态标签 (PASS/FAIL)
- 点击进入报告详情：逐步骤展示，可展开截图、AI 评语、设备日志

### 4. 用例管理 (Cases)

YAML 用例管理：
- 用例列表：名称、状态（启用/禁用）、步骤数、最后运行时间、通过率
- 搜索和排序
- 操作：新建 → 弹出 YAML 编辑器；编辑 → 内联编辑 YAML；启用/禁用开关；删除

### 5. 设备管理 (Devices)

设备状态一览：
- 卡片列表展示每个设备：名称、型号、平台、连接状态（绿/红圆点）
- 操作：刷新设备列表、连接、断开

## API 接口

```
GET  /api/stats          — 仪表盘统计数据
GET  /api/devices        — 设备列表
POST /api/devices/scan   — 扫描发现设备
GET  /api/cases          — 用例列表
GET  /api/cases/{name}   — 用例详情 (YAML 内容)
POST /api/cases          — 新建/保存用例
DELETE /api/cases/{name} — 删除用例
POST /api/run            — 运行测试
GET  /api/run/{id}/log   — SSE 实时日志流
GET  /api/reports        — 报告列表
GET  /api/reports/{id}   — 报告详情
GET  /api/reports/{id}/screenshot/{step} — 步骤截图
```

## 实现计划

### 实现文件


### 修改已有文件

- `src/gtframe/cli.py` — 增加 `web` 子命令，启动 uvicorn 服务器

### 数据交互

所有 API 返回 JSON。运行测试时使用 SSE (Server-Sent Events) 实现日志实时推送。前端 JS 通过 `EventSource` 接收日志流并追加到日志面板。

## 不包含的功能

- 用户登录/认证（单用户本地工具，无此需求）
- WebSocket 实时推送（SSE 足够）
- 数据库存储（报告从 JSON 文件读取，用例从 YAML 文件读取）
- 前端构建工具（纯 HTML/CSS/JS，零构建步骤）
